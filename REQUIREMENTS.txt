6. VARIÁVEIS DE AMBIENTE (.env)
=============================================================================

⚠️ CRITICAL - NUNCA COMMITAR ARQUIVO .env ⚠️

Backend (.env obrigatório):
---------------------------

# PostgreSQL Neon Configuration (OBRIGATÓRIO)
DATABASE_URL=postgresql://[user]:[password]@[endpoint].neon.tech/[database]?sslmode=require
   Exemplo: postgresql://cras_admin:SenhaForte123@ep-cool-fire-123456.us-east-2.aws.neon.tech/cras?sslmode=require
   
   📌 Obtenha sua URI em: https://console.neon.tech > Connection Details

DATABASE_URL_UNPOOLED=postgresql://[user]:[password]@[endpoint].neon.tech/[database]?sslmode=require
   📌 Connection string direta (sem pooling), usada para migrações Prisma

# JWT Configuration (gerar com: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))")
JWT_SECRET=[128 caracteres hexadecimais]
   Exemplo: 7b9f0d0886fba81b1102736104103188...
   ⚠️ NUNCA reutilizar em ambientes diferentes!

JWT_REFRESH_SECRET=[128 caracteres hexadecimais]
   Exemplo: 35332537be449064435d9131a20d08d4...
   ⚠️ DEVE ser diferente do JWT_SECRET!

# Encryption Configuration
ENCRYPTION_KEY=[string qualquer - NUNCA ALTERAR]
   Exemplo: seu_segredo_jwt_super_secreto_aqui_troque_em_producao
   ⚠️ CRÍTICO: Se alterar, TODOS os dados criptografados (CPF, nomes, telefones) 
                ficarão ILEGÍVEIS permanentemente! Backup obrigatório antes de qualquer mudança!

# Server Configuration
PORT=5000
NODE_ENV=development|production
FRONTEND_URL=http://localhost:5173


Frontend (.env.local opcional):
-------------------------------
VITE_API_BASE_URL=http://localhost:5000/api

Produção (Vercel - configurar no dashboard):
-------------------------------
VITE_API_BASE_URL=https://cras-agendamentos.onrender.com/api

7. CONFIGURAÇÃO DE SEGURANÇA
=============================================================================

🔒 Segurança Implementada:
---------------------------

✅ JWT com httpOnly cookies (não acessível via JavaScript)
✅ Cookies com flags: secure, httpOnly, sameSite=Strict
✅ Rate limiting (20 requests/15min por IP)
✅ Helmet.js (security headers HTTP)
✅ CORS restrito ao FRONTEND_URL
✅ Sanitização de queries SQL via Prisma (prepared statements)
✅ Criptografia AES-256-GCM para dados sensíveis (CPF, nomes, telefones)
✅ Hash bcrypt para senhas (salt rounds: 10)
✅ Validação Joi em todas as rotas
✅ Logs de auditoria com Winston
✅ DevTools detection (anti-debugging frontend)
✅ CSP (Content Security Policy) via meta tags

🔐 Dados Criptografados no Banco:
----------------------------------
✅ CPF dos usuários e agendamentos
✅ Nomes completos
✅ Números de telefone

⚠️ IMPORTANTE: A ENCRYPTION_KEY NUNCA pode ser alterada após o primeiro
   registro no banco, caso contrário TODOS os dados ficarão ilegíveis!

8. INSTALAÇÃO E DEPLOY
=============================================================================

Instalação Local:
-----------------

1. Clonar repositório:
   $ git clone [repo-url]
   $ cd CRAS-Agendamentos

2. Configurar PostgreSQL Neon (Gratuito):
   a) Acesse https://console.neon.tech e crie uma conta
   b) Crie um projeto (Free Tier)
   c) Copie as connection strings (pooled e unpooled)

3. Configurar Backend:
   $ cd backend
   $ npm install
   $ cp .env.example .env
   $ nano .env  # Configurar DATABASE_URL e DATABASE_URL_UNPOOLED
   $ npx prisma generate
   $ npx prisma db push
   $ node scripts/createAdmin.js
   $ node server.js

4. Configurar Frontend:
   $ cd ..
   $ npm install
   $ npm run dev

5. Acessar aplicação:
   Frontend: http://localhost:5173
   Backend API: http://localhost:5000


Deploy em Produção:
-------------------

1. Variáveis de Ambiente:
   ⚠️ Gerar novos secrets FORTES para JWT_SECRET e JWT_REFRESH_SECRET
   ⚠️ Configurar PostgreSQL com SSL
   ⚠️ NODE_ENV=production
   ⚠️ Configurar FRONTEND_URL com domínio real

2. Build Frontend:
   $ npm run build
   $ # Servir arquivos da pasta dist/ com nginx/apache

3. Backend em PM2:
   $ npm install -g pm2
   $ cd backend
   $ pm2 start server.js --name cras-api
   $ pm2 startup
   $ pm2 save

4. Nginx Reverse Proxy:
   server {
       listen 80;
       server_name api.seudominio.com;
       
       location / {
           proxy_pass http://localhost:5000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }

5. HTTPS com Let's Encrypt:
   $ sudo apt install certbot python3-certbot-nginx
   $ sudo certbot --nginx -d api.seudominio.com

MANUTENÇÃO E ATUALIZAÇÕES
=============================================================================

Verificar vulnerabilidades:
---------------------------
$ npm audit
$ npm audit fix

Atualizar dependências:
-----------------------
$ npm outdated
$ npm update
$ npm install <package>@latest

Backup PostgreSQL Neon:
---------------------
# Via pg_dump
$ pg_dump "postgresql://user:pass@endpoint.neon.tech/cras?sslmode=require" > backup.sql

# Ou use o backup automático do Neon (point-in-time recovery):
# Neon Console > Branches > Restore

Restore PostgreSQL Neon:
----------------------
$ psql "postgresql://user:pass@endpoint.neon.tech/cras?sslmode=require" < backup.sql
